Auto-run python scripts

Auto-running python with Rinobot

If you have some scripts which you already use for processing data, its easy to setup Rinobot to auto-trigger the script on incoming files.

Just select python from the command list, choose files to match, and select a python file to run. Currently python 3 is supported by default, but python 2 support is coming soon.

Python 3 will be called with the path of the file as an argument; so if you were to run the python script from the command line it would look like

python3 script.py /path/to/file.txt

The file path can be accessed by using the sys library:

import sys
filepath = sys.argv[1]

Full example: plotting data

In this example we make a simple XY line plot of the data in an incoming file, using numpy and matplotlib; there are both included in the Anaconda python distribution which we recommend users install when they install Rinobot.

import sys  # so that we can get the filepath
import os  # to split the filepath into name + extension
import numpy as np
import matplotlib.pyplot as plt

def main(filepath):

    # load the data
    data = np.loadtxt(filepath)

    # split into x & y and plot
    x = data[:, 0]
    y = data[:, 1:]
    plt.plot(x, y)

    # save the image as png
    filename_without_ext = os.path.splitext(filepath)[0]
    plt.savefig(filename_without_ext + '.png')


if __name__ == "__main__":
    # call the main function with the filepath
    main(sys.argv[1])

Here we use if __name__ == "__main__": so that we can still use the script in a library as needed. __name__ will only equal __main__ if the script is called from the command line
